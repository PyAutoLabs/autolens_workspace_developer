# Test Run 1 — Parametric Source-Science Recovery

Date: 2026-05-18
Issue: [PyAutoLabs/autolens_workspace_developer#72](https://github.com/PyAutoLabs/autolens_workspace_developer/issues/72)
Branch: `feature/source-science-parametric`
Worktree: `~/Code/PyAutoLabs-wt/source-science-parametric/`

## Purpose

First in a planned sequence of source-science recovery tests. The goal of
this run is to quantify how well three **parametric** source/lens models
recover the truth source magnitude and magnification on a controlled
simulation. The Euclid pipeline has shown unreliable source magnitudes
that vary with the source model — this test reproduces that pathology on
synthetic data so it can be diagnosed and fixed.

Subsequent runs (planned, not yet executed) will follow up with:

- `radius_break` re-test (eliminate `SersicCore` parameter mismatch)
- MGE with σ-cap (test the wide-Gaussian-halo hypothesis)
- `RectangularAdaptImage` pixelization fits
- `Delaunay` pixelization fits

## Setup

### Posterior expansion

Implemented in `source_science/fit_compare.py` via
`posterior_source_science_from(samples, dataset, has_linear_lp, n_draws)`:

- Draws 50 random samples from each Nautilus posterior via
  `samples.draw_randomly_via_pdf()`.
- For each sample, constructs a `Tracer` from `instance.galaxies`.
- For MGE fits, runs `al.FitImaging(dataset, tracer)` per draw and uses
  `fit.tracer_linear_light_profiles_to_light_profiles` to get the
  inversion-solved intensities. Sersic+Sersic skips this (intensities
  are non-linear, already in the posterior).
- Computes `source_science_from(tracer)` on each — image-plane flux,
  source-plane flux, magnification, magnitude — and reports median +
  asymmetric 1σ/3σ bounds, plus the truth-coverage flags
  `truth_within_1sigma`, `truth_within_3sigma`, and a z-score
  `(median - truth) / std`.

## Findings

### Finding 1 — No model recovers truth within posterior 1σ

Across 3 models × 4 derived quantities = 12 cells, exactly one
(image-plane flux for the MGE-lens / Sersic-source fit) has truth inside
the 3σ posterior band. All others fail even the 3σ test. **The
parametric-source posterior characterises data-likelihood width, not
model-truth distance.**

This is the single most important result for a Euclid analysis: any
quoted error bar on a parametric source magnitude is *systematically
under-coverage*.

### Finding 2 — Sersic+Sersic is precise but biased

Magnification 33.34 ± 0.25 vs truth 34.93 → **-5.5σ bias** (5% magnitude
of magnification, 0.067 mag of magnitude). The 1σ on magnitude is only
0.012 mag — exceptionally precise — but the truth sits seven standard
deviations away.

Two suspects for the residual bias even in the controlled case:

- **`radius_break` mismatch.** Fit uses `radius_break=0.05` on
  `SersicCore`; the simulator uses the default `0.025`. The 2× change in
  the cored inner region biases the integrated source flux. Test 2 will
  re-run with matching `radius_break` to isolate this contribution.
- **Inferred `sersic_index ≈ 1.32` vs truth `1.0`.** Higher index ⇒ more
  concentrated profile, biasing the integrated flux and the magnification
  ratio.

### Finding 3 — Lens light model is not the source-side villain

Sersic-lens → MGE-lens (both with Sersic source) moves magnification from
33.34 → 33.38 — well inside both fits' 1σ. The 5% magnification bias is
robust to lens-light model choice, so it is **a property of the source
parameterisation**, not the lens nuisance.

This is good news for Euclid: switching the lens light from Sersic to MGE
will not save you from the source-side bias. The fix must come from the
source side.

### Finding 4 — MGE source is catastrophic

Source flux: 156.7 → 318.5 (**2× over-estimate**)
Magnification: 34.93 → 19.64 (**-44% bias**)
Magnitude: 19.51 → 18.74 (**-0.77 mag**)

The MGE-source posterior is so wide (mag std 0.38, magnification std 5.4)
that this only registers as -2.0σ / -2.85σ. The posterior at least
*acknowledges* uncertainty in a way the Sersic posterior does not, but
the median is far wrong.

Working hypothesis from inspecting `fits/mge_lens__mge_source.png`:
the 40-Gaussian MGE has σ ranging from 10⁻⁴ to 3.0 ", and the inversion
can absorb diffuse residual flux into wide-σ components in the source
plane. That flux:

- inflates source-plane flux (large integration area)
- only modestly inflates image-plane flux (the diffuse component lenses
  weakly because most of it falls outside the caustics)
- therefore collapses the magnification ratio

The same pathology will appear on Euclid data, where:
1. real galaxies have non-Sersic morphology that an MGE *should* be
   better at, so users will reach for MGE
2. the MGE's flexibility is exactly what lets it eat residuals into
   non-physical haloes

### Finding 5 — Higher log-evidence ⇒ WORSE source recovery

Ranking by Nautilus log-evidence:

1. MGE + MGE source (5412.9) — **worst** source recovery
2. MGE + Sersic source (5395.3) — medium
3. Sersic + Sersic source (5381.5) — **best** source recovery

The data prefer the model that mis-measures the source most. For Euclid:
**maximising Bayesian evidence is the wrong objective if you care about
magnitude/magnification recovery.** Source magnitude is a derived
quantity; the posterior on derived quantities decouples from the
posterior on the data fit.

## Implementation notes

### Why "v3" path prefix

The Codex baseline ran fits under `source_science_v2/simple_v2/...`. A
library commit (`PyAutoGalaxy/44f8db0b "fix: give each basis independent
ell_comps in mge_model_from"`) changed the MGE model structure between
the v2 run and now, so the cached MGE samples no longer map cleanly into
the current model: `result.max_log_likelihood_fit` raises a KeyError when
walking the new model's paths against the old samples.

Bumping `path_prefix` to `source_science_v3` and `unique_tag` to
`simple_v3` makes the three Nautilus fits run fresh against the current
model definitions. The v3 outputs sit at:

```
output/output/source_science_v3/simple_v3/{sersic__sersic,mge_lens__sersic_source,mge_lens__mge_source}/
```

These directories are gitignored.

### Tutorial-script bugs caught en passant

Detailed in `source_science/TUTORIAL_NOTES.md`. Two real defects in
`autolens_workspace/scripts/imaging/features/pixelization/source_science.py`:

1. **Line 280:** `griddata(values=reconstruction)` should be
   `values=reconstruction_noise_map`. As written,
   `interpolated_noise_map` is a duplicate of the reconstruction array,
   not the noise map. Anything downstream that uses
   `interpolated_noise_map` for SNR cuts or source-plane model-fitting
   is silently wrong.
2. **Line 156 (and line 235):** `np.sum(reconstruction)` is missing the
   pixel-area factor. For a rectangular mesh the reconstruction stores
   intensity (surface brightness) per cell; the flux integral must be
   `np.sum(reconstruction * mesh_areas)`. The magnification block at
   line 311 *does* include the area factor, so the script is internally
   inconsistent.

The `autolens_workspace` repo is currently held by another task
(`profile-guide-followup-cleanup`), so these are not edited here — they
will be a follow-up issue once that workspace clears.

## Decisions and next-step priorities

1. **Test 2: `radius_break` re-test.** Re-run sersic+sersic with
   `radius_break=0.025` to match truth. Cheap, isolates one of the two
   identified bias contributors. **If this drops the magnification bias
   below 1%, the Euclid concern for parametric Sersic shrinks
   considerably.**
2. **Test 3: MGE with σ-cap.** Re-run MGE-source with maximum Gaussian
   σ capped below 1.0 " (or stronger: 0.5 "). Tests the wide-Gaussian-
   halo hypothesis. If the bias drops, the MGE problem is the halo. If
   not, it's elsewhere in the basis.
3. **Test 4: MGE source-plane regularisation.** Try a positive-flux
   prior penalty on wide-σ components, or add an explicit regularisation
   hyperparameter to the source-plane MGE.
4. **Test 5+: Pixelization.** Per the original prompt — `RectangularAdapt`
   and `Delaunay`. If those recover truth flux within uncertainty, the
   MGE pathology is specifically an artefact of the Gaussian basis. If
   they show the same bias, the problem is more general.

## Reproduce

From the worktree root with the env activated:

```bash
source ~/Code/PyAutoLabs-wt/source-science-parametric/activate.sh
cd ~/Code/PyAutoLabs-wt/source-science-parametric/autolens_workspace_developer
python source_science/fit_compare.py
```

Override `SOURCE_SCIENCE_N_DRAWS=N` to change the posterior-sample count
(default 50). The script resumes from cached Nautilus runs at
`output/output/source_science_v3/`, then redoes the posterior expansion
and writes:

- `dataset/imaging/simple/fit_comparison.json`
- `dataset/imaging/simple/fit_comparison.md`
- `dataset/imaging/simple/fits/<model_name>.png`

For a full from-scratch run (no resume), delete the v3 output directory
first. Total wall-clock: ~50 minutes (3 Nautilus fits + 50 posterior
draws each, with per-draw inversion on the two MGE fits).
